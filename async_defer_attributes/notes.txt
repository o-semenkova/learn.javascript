1) https://www.kirupa.com/html5/all_about_file_paths.htm - отличная шпаргалка по определению пути к файлу

2) Чтобы подключить несколько скриптов - используем несколько тегов подряд

3)  Кэш — промежуточный буфер с быстрым доступом, содержащий информацию, которая может быть запрошена с наибольшей вероятностью. Доступ к данным в кэше осуществляется быстрее, чем выборка исходных данных из более медленной памяти или удаленного источника, однако её объём существенно ограничен по сравнению с хранилищем исходных данных.

4) Нельзя одновременно подключить внешний скрипт и указать код (сработает код из файла)

5) Но если разбить такое подключение на 2 тега, то сработает

6) Асинхронная загрузка скрипта: async и defer (сохраняет последовательность выполнения скриптов + сработает только тогда, когда весь документ полностью загрузится) :
 
http://www.growingwiththeweb.com/2014/02/async-vs-defer-attributes.html - очень наглядно показано

7) async и defer только для внешних скриптов

8)  скрипты, добавленные через javascript, ведут себя так же, как async. То есть, выполняются как только загрузятся, без сохранения относительного порядка.
Если же нужно сохранить порядок выполнения, то есть добавить несколько скриптов, которые выполнятся строго один за другим, то используется свойство script.async = false.
